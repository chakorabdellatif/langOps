# LangOps Semantic Conventions

**Version:** 0.2.0 · **Status:** the SDK ↔ backend contract

> **0.2.0 changes (all additive — no breaking bump).** New attributes:
> `langops.node.category`; topology payload nodes may be objects
> `{id, category}` (bare strings still accepted); `langops.log` span event
> for structured logs; `langops.execution.replay_of` +
> `langops.execution.overrides` for replay lineage. Unknown attributes are
> ignored by ingestion, so a 0.1 SDK and a 0.2 backend interoperate in both
> directions.

This document defines the OpenTelemetry attribute namespace emitted by the
LangOps SDK and consumed by the `langops-api` ingestion mapper. It is the
*only* coupling between `sdk/` and `backend/`. Both sides mirror it in code:

- SDK: `sdk/src/langops/semconv.py`
- Backend: attribute constants used by `application/mappers/otlp_mapper.py`

Rules:

1. Never invent an attribute name inline — add it here first, then to the
   constants on both sides.
2. Where an official OTel GenAI convention exists (`gen_ai.*`), emit that
   name; add `langops.*` attributes only for LangGraph-specific concepts.
3. Large payloads (state snapshots, full message lists) are carried as
   **span events**, not attributes.
4. Breaking changes require a `feat(semconv)!:` commit and a version bump of
   this document.

### Payload encoding

Every payload-bearing span event (state, messages, params, response, tool
input/output, topology, execution input/output) carries its body as a JSON
string under a single event attribute, **`langops.payload`**. The backend reads
`event.attributes["langops.payload"]` and `json`-decodes it. Payloads are
redacted, then size-capped at `max_payload_bytes`; when a payload exceeds the
cap the event carries `langops.truncated = true` and `langops.payload` is a
small marker (`{"langops.truncated": true}`) instead of the oversized body.

## Resource attributes

Set once per `TracerProvider` by the SDK.

| Attribute | Type | Description |
|---|---|---|
| `service.name` | string | User's service name (from `LangOpsConfig.service_name`) |
| `langops.sdk.version` | string | SDK package version |
| `langops.project` | string | Project slug (MVP: `default`) |

## Span kinds

Every LangOps span carries `langops.kind`, which drives the ingestion mapping.

| `langops.kind` | Maps to table | Produced by |
|---|---|---|
| `execution` | `executions` | graph wrapper (root span) |
| `node` | `node_executions` | callback handler `on_chain_start/end` |
| `llm` | `llm_calls` | callback handler `on_chat_model_start` / `on_llm_end` |
| `tool` | `tool_calls` | callback handler `on_tool_start/end` |

## Execution spans (`langops.kind = execution`)

| Attribute | Type | Description |
|---|---|---|
| `langops.execution.id` | string (UUIDv7) | Generated by the SDK at run start; also stamped on all child spans |
| `langops.graph.name` | string | Graph name from config or LangGraph |
| `langops.graph.topology_hash` | string | Hash of the topology JSON (dedupe key) |
| `langops.thread.id` | string? | From `config["configurable"]["thread_id"]` |
| `langops.checkpoint.id` | string? | Checkpoint produced/ended at |
| `langops.checkpoint.parent_id` | string? | Set when resumed |
| `langops.checkpoint.resumed` | bool | Fresh vs resumed from checkpoint |

Span events:

| Event | Payload | Description |
|---|---|---|
| `langops.graph.topology` | JSON `{nodes, edges}` | Captured once via `graph.get_graph()`. **v2:** `nodes` entries may be objects `{id, category}` and `edges` entries objects `{source, target, conditional}`; bare strings / 2-tuples are still accepted |
| `langops.execution.input` | JSON | Initial graph input (truncated per limits) |
| `langops.execution.output` | JSON | Final graph output |

Replay lineage (v0.2): a replayed run's execution span additionally carries
`langops.execution.replay_of` (UUID of the original execution) and a
`langops.execution.overrides` event (JSON of the modifications applied,
redacted/truncated like any payload).

## Node spans (`langops.kind = node`)

| Attribute | Type | Description |
|---|---|---|
| `langops.execution.id` | string | Parent execution correlation (spans may arrive before the root) |
| `langops.node.name` | string | LangGraph node name |
| `langops.node.sequence` | int | Execution order within the run |
| `langops.node.retry_count` | int | Detected via repeated starts of the same node/task id |
| `langops.node.category` | string? | `llm`\|`tool`\|`utility`\|`router`\|`conditional`\|`checkpoint`\|`subgraph` (v0.2). SDK sends structural categories from the topology; the backend infers `llm`/`tool`/`utility` from child spans when absent |

Span events:

| Event | Payload | Description |
|---|---|---|
| `langops.state.input` | JSON | Node input state |
| `langops.state.output` | JSON | Node output state |
| `langops.state.diff` | JSON `{added, modified, removed}` | Structural diff (recomputed server-side; SDK may omit) |
| `langops.state.snapshot` | JSON + `langops.checkpoint.id` attrs | Reserved for the richer checkpointer snapshot in Phase 6 |

State events also carry `langops.state.size_bytes` (int) and
`langops.state.message_count` (int, when derivable) for the context-growth
series.

The checkpointer wrapper currently feeds **authoritative checkpoint lineage**
(`langops.checkpoint.id` / `.parent_id`) onto the execution span from each
`put()`; per-checkpoint state snapshot events are a Phase 6 addition.

## Structured logs (`langops.log` event, v0.2)

Emitted onto the currently-active LangOps span when log capture is enabled
(`LangOpsConfig(capture_logs=True)`). Ingestion maps each to a `logs` row, in
addition to the exception-derived rows below.

| Attribute | Type | Description |
|---|---|---|
| `langops.log.level` | string | `debug`\|`info`\|`warning`\|`error`\|`critical` |
| `langops.log.logger` | string? | Logger name |
| `langops.log.source` | string | `app`\|`sdk`\|`llm`\|`tool` (exception rows are classified `exception`) |
| `langops.payload` | JSON | `{message, extra?}` — redacted + size-capped like any payload |

## Cross-cutting

| Attribute | Type | Description |
|---|---|---|
| `langops.truncated` | bool | Present on any event whose payload was capped at `max_payload_bytes` |

Errors use standard OTel span status (`ERROR`) plus the standard
`exception` span event (`exception.type`, `exception.message`,
`exception.stacktrace`).

## Ingestion notes (backend contract)

- `span_id` / `trace_id` are the idempotency keys; ingestion is
  upsert-on-conflict.
- Child spans may arrive before their execution root; the `Execution` row is
  created lazily from `langops.execution.id` and enriched when the root lands.
- Unknown `langops.*` attributes must be ignored, never rejected (forward
  compatibility with newer SDKs).
